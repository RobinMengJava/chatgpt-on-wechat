## 一、MCP vs Skill 方案讨论

### 结论
- 决定先做 MCP，不做 Skill
- Skill 的价值在于编排多个接口成业务流程；如果按接口一对一写 Skill，不如直接用 MCP
- Spring Boot 业务接口：查询、业务操作、数据处理都有，适合 MCP

### CowAgent 当前不支持 MCP
- 项目内无任何 MCP 相关代码
- 工具加载走 `ToolManager`，只认 `BaseTool` 子类
- 做 MCP 客户端需要：MCPToolAdapter + MCPClientManager + agent_initializer 集成 + async/sync 边界处理
- 主要难点：async/sync 边界、连接生命周期管理

### 当前方案
先用 `MySQLQuery` tool 验通路，MCP 客户端作为下一阶段

## 二、MySQLQuery Tool 开发

### 新增文件
- `agent/tools/mysql/__init__.py`
- `agent/tools/mysql/mysql_query.py`
- `agent/tools/__init__.py` — 注册 MySQLQuery

### 设计要点
- 默认只读：只允许 SELECT/SHOW/DESCRIBE/EXPLAIN
- 写操作需在 config.json 设置 `mysql_allow_write: true`
- 默认返回 50 行，最多 500 行（防止撑爆上下文）
- 每次请求新建连接，查完即关闭
- 格式化输出为对齐文本表格，模型阅读友好

### config.json 新增字段
```json
"mysql_host": "your-host",
"mysql_port": 3306,
"mysql_user": "cow_reader",
"mysql_password": "yourpassword",
"mysql_database": "yourdb",
"mysql_allow_write": false,
"mysql_max_rows": 50
```

### Agent 表结构感知
- MCP Resources 可以自动注入表结构，但本质和手动写 AGENT.md 一样
- 表结构不经常变时，写在 `~/cow/AGENT.md` 完全够用
- AGENT.md 修改后需要重启服务才生效

## 三、MySQL 只读用户创建

```sql
CREATE USER 'cow_reader'@'%' IDENTIFIED BY 'yourpassword';
GRANT SELECT ON yourdb.* TO 'cow_reader'@'%';
FLUSH PRIVILEGES;
```

- `'%'` 允许任意 IP；同机部署可改为 `'localhost'`
- `FLUSH PRIVILEGES` 刷新权限缓存，用 GRANT 授权时其实会自动刷新，加不加都行

## 五、记忆系统设计讨论（per-user 隔离）

### 问题
当前 Agent 记忆系统（`~/cow/`）是全局共享的，所有用户共用同一套文件（AGENT.md、USER.md、MEMORY.md）。群聊场景下会产生用户信息互相污染的问题（如 A 让 lulu 叫她"小明"，B 也会受影响）。

### 待优化方案（参考 Claude Code 设计）

目录结构：
```
~/cow/
  AGENT.md              ← 共享，lulu 的核心人格，不可被用户覆盖
  MEMORY.md             ← 共享，群组共同知识/规则
  users/
    {session_id}/
      USER.md           ← 该用户的个人信息（称呼偏好等）
      MEMORY.md         ← 该用户的私人记忆索引
      memory/           ← 该用户的具体记忆文件
```

改动位置：`bridge/agent_initializer.py` 的 context 加载部分，在 `load_context_files(workspace_root)` 之后追加：

```python
if session_id:
    user_workspace = os.path.join(workspace_root, "users", session_id)
    if os.path.exists(user_workspace):
        user_files = load_context_files(user_workspace)
        context_files += user_files
```

核心原则：
- AGENT.md 定义 bot 本体（共享），用户无法修改
- USER.md 按 session_id 隔离，记录该用户的称呼偏好等
- 群组 MEMORY.md 存共同知识，个人 MEMORY.md 存私人记忆
- 构建 system prompt 时 = 共享层 + 用户层合并注入
