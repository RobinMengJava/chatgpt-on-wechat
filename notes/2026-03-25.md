## 二、智能客服功能实现

### 架构决策
- 知识库方案：**不引入外部服务**，使用系统内置的混合检索（向量 + 关键词，SQLite 存储）
- 知识库文件放在服务器 `~/cow/memory/knowledge/` 目录，支持多个 `.md` 文件，重启自动索引

### 新建插件 `plugins/customer_service/`
- 文件：`customer_service.py`、`config.json.template`、`__init__.py`
- 作用：挂载 `ON_DECORATE_REPLY` 事件，检测回复中包含兜底语时，将原始问题记录到日志文件
- 兜底语：`抱歉，暂时无法回答您的问题，我会记录下来，然后好好学习。`
- 未解答问题日志路径：`~/cow/unanswered_questions.log`
- **注意**：`plugins/config.json` 里的 `fallback_text` 必须与 `AGENT.md` 里写的兜底语完全一致

### 客服人设配置
- 编辑服务器上的 `~/cow/AGENT.md`，定义角色为城际大巴车票业务内部运营助手
- 关键指令：只基于知识库回答，答不上时只能回兜底语，禁止自由发挥

### .gitignore 修复
- 原本 `plugins/**/` 规则屏蔽了所有插件目录
- 新增 `!plugins/customer_service` 白名单，让新插件可以被 git 追踪

## 三、Embedding 排查过程（踩坑记录）

| 问题 | 原因 | 解决 |
|------|------|------|
| 知识库文件未被索引 | Embedding 未初始化导致 sync 失败 | 修复 embedding 后重建数据库 |
| OpenAI embedding 失败 | 使用的代理（packyapi.com）无 `text-embedding-3-small` 渠道 | 换用 LinkAI |
| `open_ai_api_base` URL 错误 | 配置了 `https://www.packyapi.com`，缺少 `/v1`，正确应为 `https://www.packyapi.com/v1` | 已修正 |
| `.env` 缓存旧 key | 系统会将 API key 迁移到 `~/.cow/.env`，已迁移的 key 不会随 config.json 更新 | 手动清空 `~/.cow/.env` 再重启 |
| LinkAI embedding 超时 | 首次请求响应慢，5s 超时 | 调通 |

### Embedding 工作机制
- 代码优先用 `open_ai_api_key`，没有再 fallback 到 `linkai_api_key`
- API key 会被自动迁移到 `~/.cow/.env`，**修改 config.json 中的 key 后需要同步更新 `.env`**，否则旧值仍然生效
- 当前使用：LinkAI embedding（`https://api.link-ai.tech/v1/embeddings`）
